API Endpoints

GET /api/v1/apps
Retrieve a list of all configured applications with their current status, configuration details, and runtime information.
GET /api/v1/apps/{app_name}
Get detailed information about a specific application including its configuration, current status (running/stopped), process ID, and uptime.
POST /api/v1/apps/{app_name}/start
Start the specified application. If the application is already running, this endpoint will return an error. Returns the new process status.
POST /api/v1/apps/{app_name}/stop
Stop the specified application gracefully. Sends a SIGTERM signal to the process and waits for it to terminate. Returns the final process status.
POST /api/v1/apps/{app_name}/restart
Restart an application by stopping it (if running) and then starting it again. Includes a brief delay between stop and start to ensure clean shutdown.
GET /api/v1/apps/{app_name}/logs
Retrieve application logs. Supports both JSON format (default) and Server-Sent Events (SSE) for real-time log streaming by setting Accept: text/event-stream header.

Usage Examples

Start an application:

curl -X POST {{.BaseURL}}/api/v1/apps/my-app/start

Stream logs in real-time:

curl -H "Accept: text/event-stream" {{.BaseURL}}/api/v1/apps/my-app/logs

Get application status:

curl {{.BaseURL}}/api/v1/apps/my-app
